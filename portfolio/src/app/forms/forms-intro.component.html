Form Basics

Angular provides two types of forms: template‐driven and reactive.

They share the same basic concepts, but have some differences concerning setup, data management, testing, etc.

Each approach offers different advantages. Selecting which one to use should be a decision of the team, but template-driven usually are not the best choice when it comes to robust forms or forms that require more complex features.

Here are some pros and cons of each one:

Reactive

favorite More scalable, reusable and testable favorite Clearer separation between business and presentation logic favorite Better support for advanced use cases favorite Can benefit from reactive transformations thumb_down Require more code thumb_down Initial setup can be harder

Template-Driven

favorite Straightforward and easier to first setup favorite Can benefit from two-way data binding thumb_down Template tends to become messy as the form grows thumb_down Less flexibility and control over the form thumb_down Some functionalities are harder to implement
