Basics

Angular provides two techniques for building forms: template‐driven and reactive.

They share the same basic concepts, but template‐driven forms are built mostly in the template, relying heavily on two‐way data binding. Reactive forms are centered in the component class.

Selecting the right approach should be a decision of the team, but in general lines, complex scenarios can be handled better with reactive forms.

Here are some pros and cons of each one:

Reactive

favorite Scalable, reusable and testable favorite Easier to perform actions on value changes favorite Easier to dynamically add input elements favorite Can benefit from reactive transformations favorite No template between code and data structures thumb_down Require more code thumb_down Harder to be implemented

Template‐Driven

favorite Straightforward and easier to be implemented favorite Very similar to HTML forms favorite Most of the work is in the template thumb_down Not as scalable, reusable or testable thumb_down Harder to use features required in complex scenarios thumb_down Less flexibility and control over the form
