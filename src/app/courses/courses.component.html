API

The following table emulates the main CRUD operations over a REST API, using an In-Memory Web Api for Angular. It allow us to simulate data persistence operations by intercepting requests that would otherwise go to a remote server. That way, no backend is needed for testing HTTP requests.

You can add a new course, edit, or delete, and then verify the updated list.

In the "Update Course" page a routed component was used instead of a dialog, intending to demonstrate both use cases.

Main Features

Route Resolvers

They were applied in some routes so that the view can only be displayed after the data from HTTP requests have been gathered. This feature can improve the user experience, since the view will not appear with empty values and have all its content displayed suddenly.

In-Memory Data Storage

Allow us to simulate data persistence operations for testing purpose, including auto-generating id's. It's a better option than a hard-coded data.

Component Communication

Table, dialog, and update components are all integrated, so the data is consistent among them, sharing changes all together.

Angular Material

The table, the dialog, and the forms were built using Angular Material components.
