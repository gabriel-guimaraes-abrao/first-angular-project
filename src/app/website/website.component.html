Website

This website was built trying to use as much as possible Angular's conventions, patterns, and best practices, along with frontend basics and web design rules, all from updated documentations and courses.

Here are some of what's been applied:

Lazy Loading and Preloading

Speed up the website's startup time, by reducing the number of files downloaded and compiled when it is launched.

Lazy loading allow us to load some modules on demand while preloading quickly launch them after the initial navigation.

The user can then benefit from a reduced startup time and a faster navigation to the lazy loaded modules.

Modular Approach

Standalone components are the new tendency in Angular apps, but since not everyone has yet migrated to the latest versions, I prefered to stick with the traditional modular approach.

The components were divided into basic application module, feature modules, and shared module.

Modules help us to organize the pieces of our application and arrange them into blocks of funcionality. Also, they are required for lazy loading.

Application Structure and Organization

The basic recommendations to keep the application organized and easy to work as it grows were applied, such as recommended file naming, folder structure, one item per file, symbol naming, organized code.

Web Design

The main design rules were applied to give the users a better experience, such as color and typography system, whitespaces and visual hierarchy concerns, optimized images, contrast checking.

Responsive Design

This website was adapted to work well in different screen sizes, by using responsive units, media queries, changing the layout of components based on screen range, and many other features.

Accessibility

Techniques, tests, and tools were applied to provide a better navigation for all kinds of users and allow people with disabilities to also have a good experience.
